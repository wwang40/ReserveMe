Overview

This document describes the REST API for ReserveMe. Use it as the single source of truth for building a React frontend (authentication flows, DTO shapes, endpoints, and examples). It assumes the backend runs on / (root) and APIs are hosted under /api.

Auth

- Authentication: JWT Bearer tokens
- Login endpoint returns a JWT; frontend stores it in memory or a secure httpOnly cookie (recommended httpOnly cookie for security).
- Protected endpoints require Authorization: Bearer <token>

Endpoints (high level)

Auth
- POST /api/auth/register
  - Request: { email, password, displayName? }
  - Response: 201 Created { id, email, displayName, createdAt }

- POST /api/auth/login
  - Request: { email, password }
  - Response: 200 { token }

Users
- GET /api/users/me
  - Auth required
  - Response: { id, email, displayName, createdAt }

Availability Slots
- POST /api/slots
  - Auth required
  - Request: { ownerId, startTime (ISO8601), endTime (ISO8601) }
  - Response: 201 { id, ownerId, startTime, endTime, createdAt }

- GET /api/slots
  - Query params: ?ownerId=<uuid> (optional)
  - Response: 200 [ { id, owner: { id, displayName }, startTime, endTime, createdAt } ]

Reservations
- POST /api/reservations
  - Auth required
  - Request: { slotId, userId, notes? }
  - Response: 201 { id, slotId, userId, createdAt }

- GET /api/reservations?userId=<uuid>
  - Auth required
  - Response: 200 [ { id, slot: { id, startTime, endTime }, user: { id, displayName }, createdAt } ]

DTO shapes
- UserResponse: { id: UUID, email: string, displayName: string, createdAt: string }
- AvailabilitySlotResponse: { id: UUID, owner: { id: UUID, displayName: string }, startTime: string, endTime: string, createdAt: string }
- ReservationResponse: { id: UUID, slot: AvailabilitySlotResponse, user: UserResponse, createdAt: string }

Frontend guidance for Copilot

- Use React + TypeScript.
- Store token in an httpOnly cookie (backend should set cookie) or in memory. Avoid localStorage for JWT if possible.
- Provide an AuthContext that exposes: user, token, login(email,password), logout(), register(...).
- Use Axios or fetch with an interceptor that attaches Authorization header.
- Create components: LoginForm, RegisterForm, SlotList (with optional owner filter), SlotCreate, ReservationList, ReservationCreate.

Security notes for frontend

- Always include CSRF protections if storing token in cookies. Prefer double-submit cookie or same-site cookies.
- Validate time inputs on the frontend before sending to backend.

Example API calls (fetch)

- Login
  fetch('/api/auth/login', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ email, password }) })

- Get slots
  fetch('/api/slots').then(res => res.json())

- Create slot (requires token)
  fetch('/api/slots', { method: 'POST', headers: { 'Content-Type': 'application/json', 'Authorization': `Bearer ${token}` }, body: JSON.stringify({ ownerId, startTime, endTime }) })

Notes for Copilot usage

- Use the DTO shapes above when creating API service functions.
- Prefer async/await and centralize error handling.
- Mocked data can be used for initial UI skeleton; wire to real endpoints later.
